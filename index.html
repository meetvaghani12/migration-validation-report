Deduplication & Migration Decision

Content-Based Duplicate Analysis

Using ETag + file size fingerprinting (zero egress cost), 944,346 duplicate groups were identified across 3,413,684 file entries in the source bucket. Each file was cross-referenced against live CDN URLs from 4 CREA-DESI project files.

Action Classification & Migration Decision
ActionFilesStorageDecisionReason
KEEP 936,886 5,646 GB Migrated Original copy — earliest-written file in each duplicate group. Must be preserved.
DO_NOT_DELETE 70,038 58 GB Migrated Duplicate files whose exact OSS path is referenced by live CDN URLs from client's DAM system. Migrated to preserve CDN link integrity.
SAFE_TO_DELETE 2,406,760 9,010 GB Not Migrated Duplicate files with no CDN reference. Content is byte-for-byte identical to KEEP originals. Safely excluded.
Why DO_NOT_DELETE files were migrated: These 70,038 files are duplicates (content also exists via KEEP originals), but their exact paths are actively referenced by CDN URLs across CREA-DESI web properties. They were migrated to ensure all CDN-referenced content is available in the new bucket, preserving link integrity.
Why SAFE_TO_DELETE files were not migrated: These 2,406,760 files are byte-for-byte identical copies of KEEP originals with zero active CDN references. Their content is fully preserved through the migrated KEEP files. The migration script used duplicates_with_usage_20260324_023501.csv.gz (Reference #3) to build the skip list, filtering on action == SAFE_TO_DELETE.
