Source Analysis & Deduplication

Duplicate Detection

A content-hash approach (ETag + file size) identified all duplicate files across the source bucket — zero file downloads, zero egress cost. Files sharing identical (size, ETag) pairs were grouped. The earliest-written copy in each group was designated as the original.

944,346 duplicate groups spanning 3,413,684 file entries were identified.

Action Classification
ActionFilesStorageDecision
KEEP 936,886 5,646.27 GB Original copy — must be preserved
DO_NOT_DELETE 70,038 58.05 GB Duplicate but referenced by live CDN URL
SAFE_TO_DELETE 2,406,760 9,009.89 GB Duplicate — no CDN reference. Excluded from migration.
The migration script consumed duplicates_with_usage_20260324_023501.csv.gz directly, filtering action == SAFE_TO_DELETE to build the skip set. Only KEEP and DO_NOT_DELETE files were migrated.

Conclusion

Migration Complete

The migration from pimcore-prod-01 (Singapore) to pimcore-prod-assets (Dubai) has been completed with comprehensive verification:

  1. Deduplication reduced 3.4M file entries to unique files by safely excluding 2.4M duplicates
  2. Path restructuring transformed the deep folder hierarchy into a flat {uid}/{filename} scheme
  3. Multi-VM parallel migration transferred all files to the Dubai region
  4. Double verification confirmed 100% match rate with 0 size mismatches
  5. Ongoing sync ensures the target bucket stays current with source changes
